七牛云存储 JS-SDK

简单、快捷上传至七牛云存储的网页端解决方案

概述

功能

  • 浏览器端直接上传文件至七牛云存储，并提供下载链接
  • 支持为每个文件配置单独的上传 token
  • 支持自定义上传域名，自动匹配合适的上传域名(https 或 http)
  • 在 HTML5 模式下，大于 4M 的文件支持分块上传、断点续上传
  • 支持 HTML5、Flash、HTML4 依次退化上传
  • 继承了所有 Plupload 插件的功能

依赖

  • JS-SDK 依赖于 Plupload 插件，必须先引入 Plupload
  • JS-SDK 依赖服务端提供上传 token

不推荐场景

  • 低版本的IE（IE6/7）。虽然理论上 JS-SDK 支持低版本IE，但并未充分测试，所以可能出现兼容性问题
  • 专门面向移动端的网站。Plupload 插件太重，推荐选择更轻量的上传插件或者自行开发

2、服务端准备

服务端需要基于七牛云存储提供的 ACCESS_KEY / SECRET_KEY（即AK/SK，点击获取）生成用于上传的 token 。

开发者可以依赖七牛官方提供的各种服务端 SDK生成，也可以自己基于七牛文档-上传策略开发。

以下是用七牛 node-js 生成 token 的简单示例代码：


      var qiniu = require('qiniu');
      var express = require('express');
      var app = express();

      qiniu.conf.ACCESS_KEY = 七牛 ACCESS_KEY ;
      qiniu.conf.SECRET_KEY = 七牛 SECRET_KEY ;

      var bucket_name = 七牛空间名字 ;
      var uptoken = new qiniu.rs.PutPolicy(bucket_name);

      app.get('/uptoken', function(req, res, next) {
          var token = uptoken.token();
          if (token) {
              res.json({
                  uptoken: token
              });
          }
      });

注意： 上例中的 '/uptoken' 是客户端请求的 URL，用于赋值给 JS-SDK 的初始化参数 'uptoken_url'

服务端提供的 token 通过 JSON 格式返回给浏览器端，格式约定如下：


      {
        你的上传token
      }
或 

      {
        'uptoken': 你的上传token
      }

运行示例

  • 安装 Nodejs

  • 获取源代码： git clone git@github.com:qiniupd/qiniu-js-sdk.git

  • 在根目录运行npm install，安装七牛 Node.js SDK、Express

  • 进入demo目录,修改config.js，Access Key和Secret Key 按如下方式获取

    • 开通七牛开发者帐号

    • 登录七牛开发者自助平台，查看 AccessKey 和 SecretKey 。

            
        module.exports = {
            'ACCESS_KEY': '',
            'SECRET_KEY': '',
            'Public_Bucket_Name': 'qiniu-plupload',
            'Public_Bucket_Domain': '',
            'Public_Uptoken_Url': '/uptoken',
            'Port': 19110,
            'Private_Bucket_Name': 'plupload-private',
            'Private_Bucket_Domain': '',
            'Private_Uptoken_Url': '/private_uptoken'
        };

  • 在demo目录运行node server.js 或者 在根目录运行make启动

  • 访问http://127.0.0.1:18080/或http://localhost:18080/
